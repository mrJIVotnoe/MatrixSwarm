# RFC: E.S.C.A.P.E. Client Architecture
**End-User Symbiotic Client for Autonomous Packet Evasion**

## 1. Введение
E.S.C.A.P.E. — это клиентское приложение, устанавливаемое на устройствах пользователей (Граждан). Оно выполняет две основные функции:
1. **Локальный прокси-сервер**: Модифицирует сетевые пакеты для обхода систем глубокого анализа трафика (DPI).
2. **Симбионт Роя**: Поддерживает связь с C2-сервером (Ульем), получает оптимальные стратегии обхода для своего провайдера и отправляет анонимную телеметрию (Waggle Dance).

## 2. Технологический стек (Гибридная архитектура)
Для достижения максимальной производительности, безопасности и скорости разработки выбрана гибридная архитектура:

*   **Go (Golang)**: Ядро управления (The Brain). Отвечает за высокоуровневую логику, сеть, связь с C2, локальный HTTP/SOCKS5 сервер и управление конфигурацией.
*   **Rust**: Движок обхода (The Muscle). Отвечает за низкоуровневую манипуляцию пакетами (TCP/TLS/HTTP) на лету. Вызывается из Go через FFI (Foreign Function Interface) с использованием C-bindings.

## 3. Компоненты системы

### 3.1. Go Core (Управляющее Ядро)
*   **Swarm Communicator**: Модуль связи с Ульем. Выполняет регистрацию узла, отправляет Heartbeat-запросы, получает задачи и стратегии от Комиссара.
*   **Telemetry Agent**: Собирает данные о задержках (latency) и успешности соединений. Отправляет анонимные отчеты (без PII, только хэш ISP и метрики).
*   **Local Proxy Listener**: Поднимает локальный SOCKS5/HTTP прокси (например, на `127.0.0.1:1080`), на который пользователь заворачивает свой трафик.
*   **Strategy Manager**: Парсит параметры стратегии, полученные от Улья (например, `--split-pos 3 --diso --fake-sni`), и передает их в Rust-движок.

### 3.2. Rust Engine (Движок Эвазии)
*   **Packet Mangler**: Перехватывает байты, идущие от Local Proxy Listener к целевому серверу.
*   **DPI Strategies**:
    *   *Split*: Разделение ClientHello на несколько TCP-сегментов.
    *   *Disorder (Diso)*: Отправка сегментов в неправильном порядке (с использованием сырых сокетов или манипуляций на уровне прокси).
    *   *Fake*: Отправка поддельного пакета с низким TTL перед настоящим.
    *   *HostCase*: Изменение регистра заголовка Host (для HTTP).
*   **C-API (FFI)**: Экспортирует функции для Go, например: `int process_stream(int fd, const char* strategy_config)`.

## 4. Жизненный цикл (Data Flow)

1.  **Инициализация**: Пользователь запускает E.S.C.A.P.E. Go Core определяет локального провайдера (ISP) через публичный IP/BGP ASN.
2.  **Симбиоз**: Go Core регистрируется в Улье (`/api/v1/nodes/register`).
3.  **Получение Директивы**: Go Core отправляет Heartbeat. Комиссар Улья отвечает оптимальной стратегией для данного ISP.
4.  **Настройка Движка**: Go Core инициализирует Rust Engine полученными параметрами.
5.  **Трафик**: Браузер пользователя отправляет запрос на `127.0.0.1:1080`. Go Core принимает соединение и передает поток в Rust Engine.
6.  **Эвазия**: Rust Engine модифицирует пакеты (например, дробит TLS ClientHello) и отправляет их целевому серверу.
7.  **Танец Пчел (Телеметрия)**: Go Core замеряет время ответа и успешность рукопожатия (handshake). Данные отправляются в Улей (`/api/v1/nodes/{id}/tasks/{id}/complete`).

## 5. Безопасность и Приватность (Протокол Омега)
*   **Zero PII**: Клиент никогда не отправляет историю браузера, IP-адреса назначения (если это не тестовые пинги Роя) или личные данные.
*   **Эфемерность**: В случае обнаружения компрометации (например, попытки реверс-инжиниринга или блокировки C2), клиент активирует "Протокол Омега" — стирает локальные кэши стратегий и переходит в спящий режим.
*   **Шифрование C2**: Вся связь с Ульем идет через HTTPS с pinning'ом сертификатов (или дополнительным слоем шифрования поверх HTTP).

## 6. План разработки (Фаза 2)
1.  **Шаг 1**: Создание каркаса Go-приложения (CLI, конфигурация, локальный SOCKS5).
2.  **Шаг 2**: Реализация связи с C2 (API клиент для регистрации и Heartbeat).
3.  **Шаг 3**: Создание Rust-библиотеки (FFI) для базового сплиттинга пакетов.
4.  **Шаг 4**: Интеграция Go и Rust (передача TCP-потока из Go в Rust).
5.  **Шаг 5**: Сбор и отправка телеметрии обратно в Улей.
